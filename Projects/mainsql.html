Complex Data Extraction

This project involved developing complex SQL queries for extracting data from multiple related tables in a PostgreSQL database. The solution optimized query performance by 50% through strategic indexing and query restructuring.

-- Complex multi-table join with subqueries
SELECT 
    c.customer_id,
    c.customer_name,
    o.order_date,
    o.total_amount,
    p.product_name,
    p.category
FROM customers c
JOIN orders o ON c.customer_id = o.customer_id
JOIN order_items oi ON o.order_id = oi.order_id
JOIN products p ON oi.product_id = p.product_id
WHERE o.order_date >= '2023-01-01'
    AND c.customer_id IN (
        SELECT customer_id 
        FROM orders 
        WHERE total_amount > (
            SELECT AVG(total_amount) 
            FROM orders
        )
    )
ORDER BY o.order_date DESC;

  • Complex Joins: Successfully implemented 4-table joins with subqueries

Developed a comprehensive ETL pipeline using SQL Server Integration Services (SSIS) to process over 1 million records daily. The pipeline includes data validation, transformation, and loading processes with automated error handling.

-- ETL Stored Procedure for Data Processing
CREATE PROCEDURE sp_ProcessDailyData
AS
BEGIN
    SET NOCOUNT ON;
    
    -- Data validation and cleaning
    UPDATE staging_table 
    SET email = LOWER(TRIM(email))
    WHERE email IS NOT NULL;
    
    -- Transform and load data
    INSERT INTO fact_sales (customer_id, product_id, sale_date, amount)
    SELECT 
        c.customer_id,
        p.product_id,
        s.sale_date,
        s.amount
    FROM staging_sales s
    JOIN dim_customers c ON s.customer_email = c.email
    JOIN dim_products p ON s.product_sku = p.sku
    WHERE s.sale_date >= DATEADD(day, -1, GETDATE());
    
    -- Update statistics
    UPDATE STATISTICS fact_sales;
END;

Analytics Dashboard Queries

×

Created real-time analytics queries for a Tableau dashboard serving 200+ daily active users. The queries provide insights into user behavior, performance metrics, and business KPIs with sub-second response times.

-- Real-time analytics query for dashboard
WITH user_metrics AS (
    SELECT 
        DATE(created_at) as date,
        COUNT(*) as total_users,
        COUNT(CASE WHEN status = 'active' THEN 1 END) as active_users
    FROM users
    WHERE created_at >= DATE_SUB(NOW(), INTERVAL 30 DAY)
    GROUP BY DATE(created_at)
),
daily_performance AS (
    SELECT 
        DATE(order_date) as date,
        COUNT(*) as orders,
        SUM(total_amount) as revenue,
        AVG(total_amount) as avg_order_value
    FROM orders
    WHERE order_date >= DATE_SUB(NOW(), INTERVAL 7 DAY)
    GROUP BY DATE(order_date)
)
SELECT 
    um.date,
    um.total_users,
    um.active_users,
    dp.orders,
    dp.revenue,
    dp.avg_order_value,
    ROUND((um.active_users / um.total_users) * 100, 2) as activation_rate
FROM user_metrics um
LEFT JOIN daily_performance dp ON um.date = dp.date
ORDER BY um.date DESC;

Dashboard Features

  • Real-time Data: Sub-second query response times for live dashboards
  • User Analytics: Comprehensive user behavior and engagement metrics
  • Performance KPIs: Revenue, conversion rates, and business metrics
  • Scalability: Optimized for 200+ concurrent users

Data Security & Access Control

×

Implemented comprehensive data security measures using Oracle PL/SQL to ensure GDPR and SOX compliance. The solution includes role-based access control, data encryption, and audit logging for sensitive data protection.

-- Security implementation with role-based access
CREATE OR REPLACE PROCEDURE secure_data_access(
    p_user_id IN VARCHAR2,
    p_table_name IN VARCHAR2,
    p_operation IN VARCHAR2
) AS
    v_user_role VARCHAR2(50);
    v_has_access BOOLEAN := FALSE;
BEGIN
    -- Get user role
    SELECT role_name INTO v_user_role
    FROM user_roles 
    WHERE user_id = p_user_id;
    
    -- Check access permissions
    SELECT COUNT(*) INTO v_has_access
    FROM role_permissions rp
    JOIN table_permissions tp ON rp.permission_id = tp.permission_id
    WHERE rp.role_name = v_user_role
        AND tp.table_name = p_table_name
        AND tp.operation = p_operation;
    
    IF v_has_access = 0 THEN
        RAISE_APPLICATION_ERROR(-20001, 'Access denied: Insufficient permissions');
    END IF;
    
    -- Log access attempt
    INSERT INTO access_log (user_id, table_name, operation, access_time)
    VALUES (p_user_id, p_table_name, p_operation, SYSDATE);
    
    COMMIT;
END;

Security Features

  • Role-Based Access: Granular permissions based on user roles and responsibilities
  • Data Encryption: AES-256 encryption for sensitive data at rest and in transit
  • Audit Logging: Comprehensive tracking of all data access and modifications
  • Compliance: Full GDPR and SOX compliance with automated reporting

-- Performance optimization with strategic indexing
-- Before: 2.5 seconds execution time
-- After: 0.6 seconds execution time

-- Create composite index for frequently queried columns
CREATE INDEX idx_orders_customer_date 
ON orders (customer_id, order_date, status);

-- Optimize query with proper index usage
SELECT 
    o.order_id,
    o.order_date,
    o.total_amount,
    c.customer_name,
    p.product_name
FROM orders o
INNER JOIN customers c ON o.customer_id = c.customer_id
INNER JOIN order_items oi ON o.order_id = oi.order_id
INNER JOIN products p ON oi.product_id = p.product_id
WHERE o.order_date >= '2023-01-01'
    AND o.status = 'completed'

        SELECT customer_id 
        FROM orders 
        WHERE total_amount > 1000
    )

Data Warehouse Design

×

Designed and implemented a comprehensive data warehouse solution using SQL Server and SSAS, processing over 10TB of data with star schema design and automated ETL processes for business intelligence reporting.

-- Data Warehouse ETL Process
-- Fact table creation with proper indexing
CREATE TABLE fact_sales (
    sales_key INT IDENTITY(1,1) PRIMARY KEY,
    customer_key INT NOT NULL,
    product_key INT NOT NULL,
    date_key INT NOT NULL,
    sales_amount DECIMAL(18,2),
    quantity INT,
    discount_amount DECIMAL(18,2),
    FOREIGN KEY (customer_key) REFERENCES dim_customers(customer_key),
    FOREIGN KEY (product_key) REFERENCES dim_products(product_key),
    FOREIGN KEY (date_key) REFERENCES dim_date(date_key)
);

-- ETL stored procedure for data loading
CREATE PROCEDURE sp_LoadSalesData
AS
BEGIN
    SET NOCOUNT ON;
    
    -- Truncate staging table
    TRUNCATE TABLE staging_sales;
    
    -- Load data from source systems
    INSERT INTO staging_sales (customer_id, product_id, sale_date, amount, quantity)
    SELECT 
        c.customer_id,
        p.product_id,
        s.sale_date,
        s.amount,
        s.quantity
    FROM source_sales s
    INNER JOIN source_customers c ON s.customer_id = c.customer_id
    INNER JOIN source_products p ON s.product_id = p.product_id
    WHERE s.sale_date >= DATEADD(day, -1, GETDATE());
    
    -- Transform and load into fact table
    INSERT INTO fact_sales (customer_key, product_key, date_key, sales_amount, quantity)
    SELECT 
        dc.customer_key,
        dp.product_key,
        dd.date_key,
        ss.amount,
        ss.quantity
    FROM staging_sales ss
    INNER JOIN dim_customers dc ON ss.customer_id = dc.customer_id
    INNER JOIN dim_products dp ON ss.product_id = dp.product_id
    INNER JOIN dim_date dd ON ss.sale_date = dd.full_date;
    
    -- Update statistics
    UPDATE STATISTICS fact_sales;
END;

Data Warehouse Features

  • Star Schema Design: Implemented optimized star schema for fast analytical queries
  • ETL Automation: Automated data processing pipeline handling 10TB+ of data
  • Performance: Sub-second response times for complex analytical queries
  • Scalability: Designed to handle growing data volumes with horizontal scaling

Graduation Survey Analysis

×

Developed a comprehensive SQL query using Common Table Expressions (CTEs) to analyze graduation survey responses across multiple cohorts. The query joins student degree data with survey responses to provide insights into student outcomes and engagement patterns.

WITH
_Grad
AS
(
SELECT d.PersonID
      ,sem.AcademicReportingYear
   ,'20'+LEFT(sem.AcademicReportingYear, 2) + '-' + '20'+RIGHT(sem.AcademicReportingYear,2) AS DisplayAcademicYear
      ,sem.Year
      ,sem.SemesterOrderRelativeToCurrent
      ,sem.TermCode
      ,m.College
      ,m.Department
      ,m.MajorName
FROM dbo.Degree AS d
INNER JOIN dbo.Semester AS sem ON sem.SemesterID = d.DegreeGrantSemesterID
INNER JOIN dbo.Major AS m ON m.MajorID = d.Major1ID
WHERE d.Subprogram = 'Day School (DAY)'
  AND d.IsBYUIDegree = 1
  AND m.Level = 'B'
  AND sem.Session <> 'Summer Session'
)
,
_SurveyResponse
AS
(
SELECT sqr.PersonID
      ,sr.Cohort
FROM dbo.factSurveyQuestionResponse AS sqr
INNER JOIN dbo.dimSurveyQuestion AS sq ON sqr.dimSurveyQuestionID = sq.dimSurveyQuestionID
INNER JOIN dbo.dimSurveyResponseInfo AS sr ON sqr.dimSurveyResponseInfoID = sr.dimSurveyResponseInfoID
WHERE sr.Cohort IN ('5 Year', '120 Days', '1 Year', 'Graduation')
GROUP BY sqr.PersonID
        ,sr.Cohort
)
SELECT g.AcademicReportingYear
   ,g.DisplayAcademicYear
      ,g.Year
      ,g.SemesterOrderRelativeToCurrent
      ,g.TermCode
      ,g.College
      ,g.Department
      ,g.MajorName
      ,count (DISTINCT (CASE WHEN s.Cohort = '5 Year' THEN g.PersonID ELSE NULL END)) '5 Year Cohort'
      ,count (DISTINCT (CASE WHEN s.Cohort = '120 Days' OR s.Cohort = 'Graduation' THEN g.PersonID ELSE NULL END)) '120 Days Cohort'
      ,count (DISTINCT (CASE WHEN s.Cohort = '1 Year' THEN g.PersonID ELSE NULL END)) '1 Year Cohort'
      ,count (DISTINCT (CASE WHEN s.Cohort = 'Graduation' THEN g.PersonID ELSE NULL END)) 'Graduation Cohort'
      ,count (DISTINCT g.PersonID) Denominator
FROM _Grad AS g
LEFT OUTER JOIN _SurveyResponse AS s ON s.PersonID = g.PersonID
GROUP BY g.AcademicReportingYear
  ,g.DisplayAcademicYear
        ,g.Year
        ,g.SemesterOrderRelativeToCurrent
        ,g.TermCode
        ,g.College
        ,g.Department
        ,g.MajorName;

Query Features

  • Common Table Expressions: Used CTEs for complex data transformation and organization
  • Multi-table Joins: Integrated degree, semester, major, and survey data
  • Cohort Analysis: Analyzed student responses across different time periods (5 Year, 120 Days, 1 Year, Graduation)
  • Conditional Aggregation: Used CASE statements for cohort-specific counting
  • Data Filtering: Applied multiple WHERE conditions for data quality and relevance
  • Academic Reporting: Generated display-friendly academic year formatting
